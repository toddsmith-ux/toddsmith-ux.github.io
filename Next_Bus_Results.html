Contextual Inquiry

After creating the prototype, we conducted a small contextual inquiry to survey users about their impressions of NextBus. Out of the users we surveyed, three of them were active NextBus users and one was new to the platform. Each user met face-to-face with a team member, and were shown the existing NextBus mobile application.

First, we asked them about their general impressions of the application. Then, we asked the users about issues and desired changes to existing functionality. Next we asked users to complete the following tasks:

  • 1. Find specific route and accompanying map
  • 2. Edit application settings
  • 3. Set a frequent destination

Afterwards, we showed the users our prototype and asked them about their impressions. Since this is a small study, we acknowledge that we cannot draw any generalizations from this test. From the contextual inquiry, we learned the following insights: Strengths of the redesign include the simplification of navigation flow and the comprehensiveness of user profile

Weaknesses included the order of menu buttons and the lack of support features for new users. For individuals who were active NextBus users, their existing preferences and habits affected their statements. As an example, combining the map view and list view to form the primary navigation flow was highly supported by the comments of some active users, while others did not find any improvement as they did not use the map at all.

Conclusion

From this experience, I believe the changes that we implemented will improve usability for the user. The most important lesson I learned from this experience is the value of cognitive load. When users navigate a user interface, they only need to know the see the essential information. Thus, cognitive load can make or break a userâ€™s experience.
