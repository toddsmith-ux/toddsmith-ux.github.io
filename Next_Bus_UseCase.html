User Task Analysis

Use Cases

To outline the user's process as they navigate through the interface, we wrote use cases for the following tasks:

  1. Find the next available bus to take the user home (via multiple bus agencies)
  2. Create a user profile (to indicate home, favorites, and frequent locations)
  3. Open help menu for navigation

We also created hierarchical task analyses for each use case. For brevity, I will only outline one use case.

Use Case 1 is arguably the most frequent task. Thus, this use case has a significant amount of sub tasks.

Use Case 1

A: The user opens NextBus.

  1. The user opens the application.
  2. The user finds the app on their phone.
  3. The user selects the app.

B: The user plans route.

  1. The user finds transit options.
  2. The user selects Transit menu option.
  3. The user selects the agency (one or more than one).
  4. The user selects the route.
  5. The user specifies the destination.
  6. The user specifies the stop.
  7. The user finds nearby options.
  8. The user selects the nearby menu option.
  9. The user filters through nearby menu options in the direction of their destination.
  10. The user selects the ideal route.
  11. The user finds the fastest route.
